Normative Assertions Summary

td-string-type: MUST
Vocabularies that use simple types string and anyURI MUST be serialized as JSON string.
  • TO DO: test spec for td-string-type. TO DO: test spec for td-integer-type.
td-integer-type: MUST
Vocabularies that use simple type unsignedInt MUST be serialized as JSON integer.
  • TO DO: test spec for td-integer-type.
td-number-type: MUST
Vocabularies that use simple type double MUST be serialized as JSON number.
td-context: MAY
In the first line of the Thing Description instance MAY include the @context key from JSON-LD 1.1 with the value URI of the Thing description context file https://w3c.github.io/wot/w3c-wot-td-context.jsonld.
td-context-jsonld: MUST
In the case the Thing Description instance is processed and interpret by a JSON-LD 1.1 processor this @context with its value MUST be present
td-additional-contexts: MUST
In case a single Thing Description instance involves several contexts, additional namespaces with prefixes MUST be extended to the @context array structure.
td-context-toplevel: MUST
Each mandatory and optional vocabulary, as defined in the class Thing, MUST be serialized as a JSON key at the top level of the Thing Description instance.
td-objects: MUST
The type of the fields properties, actions, and events MUST be serialized as a JSON object.
td-arrays: MUST
The type of the field links and security MUST be serialized as a JSON array.
td-properties: MUST
Properties (and sub-properties) offered by a Thing MUST be collected in the JSON object-based properties field of (unique) Property names as JSON keys.
td-property-keys: MUST
Each mandatory and optional vocabulary, as defined in the class Property, its two superclasses InteractionPattern and DataSchema, MUST be serialized as a JSON key within a Property field.
td-property-objects: MUST
The type of the fields properties and items MUST be serialized as a JSON object.
td-property-arrays: MUST
The type of the fields forms, required, and enum MUST be serialized as a JSON array.
td-property-semantic: MAY
Similar to the Thing level properties MAY have additional semantic anotations based on JSON-LD 1.1 keys.
td-property-defaults: MUST
In the case the Thing Description instance is processed and interpret by a JSON-LD 1.1 processor, each property MUST contain the vocabularies observable and writable due to the linked data's open-world assumption.
td-actions: MUST
Each optional vocabulary, as defined in the class Action and its superclass InteractionPattern, MUST serialized as a JSON key within a Action field.
td-action-objects: MUST
The type of the fields input and output MUST be serialized as a JSON object.
td-action-not-objects: MUST NOT
The forms field MUST NOT be serialized as a JSON object.
td-action-arrays: MUST
The type of the field forms MUST be serialized as a JSON array.
td-action-semantic: MAY
Definitions within the events level MAY have also additional semantic anotations based on JSON-LD 1.1 keys.
td-events: MUST
Each optional vocabulary, as defined in the class Event its two superclasses InteractionPattern and DataSchema, MUST be transformed as a JSON key within a Event field.
td-event-objects: MUST
The type of the fields properties and items MUST be serialized as a JSON object.
td-event-arrays: MUST
The type of the fields forms, required, and enum MUST be serialized as a JSON array.
td-forms: MUST
Each mandatory and optional vocabulary, as defined in the class Form, MUST be serialized as a JSON key.
td-form-protocolbindings: MAY
If required, forms MAY be supplemented with protocol-specific vocabularies identified with a prefix. See also [[!WOT-PROTOCOL-BINDING]].
td-form-mediatype: MUST
In the case the Thing Description instance is processed and interpret by a JSON-LD 1.1 processor, each forms MUST be contain the vocabulary mediaType due to the linked data's open-world assumption.
td-links: MUST
Each mandatory and optional vocabulary, as defined in the class Link, MUST be serialized as a JSON key.
td-security: MUST
Each mandatory and optional vocabulary, as defined in the class SecurityScheme, MUST be serialized as a JSON key.
td-jsonld-preprocessing-context: MUST
There MUST be a @context key from JSON-LD 1.1 as defined in Section Thing as a whole.
td-jsonld-preprocessing-prefix: MUST
External used vocabularies in the Thing Descriptions MUST provide their context URIs as prefix or within the @context.
td-jsonld-preprocessing-defaults: MUST
All vocabularies that have a default value as defined Vocabulary Definition MUST be present at least with their default value.
